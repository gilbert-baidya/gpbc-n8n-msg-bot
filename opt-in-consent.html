Opt-In Method: User-Initiated Contact

How Consent is Collected

Our SMS information service operates on a strict opt-in basis through user-initiated contact. Users must actively text our church number to receive any automated responses.

Key Points:

  • ✓ Users must send the first message to opt-in
  • ✓ No unsolicited messages are ever sent
  • ✓ No bulk messaging or marketing campaigns
  • ✓ Responses only sent when user texts a command
  • ✓ Users can opt-out anytime by texting "STOP"

How the System Works

Our automated SMS bot is a response-only system. It cannot initiate conversations or send unsolicited messages. The system only responds when someone texts specific keywords to our church number.

Available Commands

  • hi or hello - Receive welcome message and information about available commands
  • service - Get church service times and location
  • prayer - Request prayer support
  • jesus - Learn about salvation through Jesus Christ
  • pastor - Get pastor contact information
  • help - See list of all available commands
  • stop or unsubscribe - Opt-out of receiving any further messages

Consent Process

By texting any keyword to our church number, users explicitly consent to receive automated information responses. This constitutes opt-in consent because:

  1. Voluntary Action: Users must take the initiative to text us first
  2. Informed Participation: Users receive a welcome message explaining how the system works
  3. Easy Opt-Out: Users can text "STOP" at any time to unsubscribe
  4. No Commercial Intent: Messages are purely informational about church services

Data Privacy

We respect user privacy:

  • Phone numbers are not stored in any database
  • No user data is collected or retained
  • Messages are processed in real-time and not logged
  • No third-party sharing of any information

Message Frequency

Users only receive messages when they actively text a command. There are no scheduled messages, newsletters, or promotional texts. Message frequency is entirely controlled by the user.

Technical Implementation

The system is implemented as a webhook-based response bot that:

  • Only processes incoming SMS messages
  • Has no capability to initiate outbound messages
  • Cannot send bulk or broadcast messages
  • Responds only to recognized keywords

Source Code: The complete implementation is publicly available at https://github.com/gilbert-baidya/gpbc-n8n-msg-bot

Compliance

This SMS service complies with:

  • TCPA (Telephone Consumer Protection Act) requirements
  • CTIA Messaging Principles and Best Practices
  • Twilio's Messaging Policy
  • User-initiated opt-in consent standards
